# Life vs. Global Timeline Visualizer

A highly customizable, dual-axis web application that maps personal life events against global macro-historical events. It provides a unique visual perspective on how world history intersects with individual milestones.

The application uses a purely client-side architecture (HTML/CSS/JS) meaning no data is sent to external servers. Events are parsed from a structured text format and rendered dynamically into a responsive, color-coded timeline.

# 🚀 Quick Start

No build tools (Node, npm, Webpack) are required, and no local server is needed.

Open `demo/index.html` directly in any modern web browser (Chrome, Firefox, Safari, Edge). A sample dataset is already pre-loaded in the textarea, so the timeline renders immediately — edit the text and click **Generate Timeline** to visualize your own data.

# 📝 How to Format Data

The timeline is generated by parsing a specific text structure inside the textarea. Use the following format for each event:

```
DATE: [YYYY.MM.DD] | TYPE: [LOCAL/GLOBAL] | CAT: [Category] | TEXT: [Event Description]
```

Example:

```
DATE: 1989.11.09 | TYPE: GLOBAL | CAT: politics | TEXT: Fall of the Berlin Wall
```

Notes on the format:

- **Date order is year.month.day (ISO-like).** Earlier versions split the year into its own field and used a `DD.MM` date, which is ambiguous to parse and sort correctly. `YYYY.MM.DD` removes that ambiguity — the year no longer needs a separate field, since it's derived directly from the date.
- **Invalid lines are skipped, not fatal.** A line that doesn't match the format is ignored and logged (with its line number) to the browser console (DevTools → Console) so you can find and fix it.
- **Category names are restricted** to lowercase letters, numbers, `_` and `-`, since each category name is used directly as a CSS class. Anything else falls back to a generic `other` category rather than being rejected outright.

# Supported Categories

Categories use a flat, lowercase, single-word taxonomy designed to be mutually exclusive (an event shouldn't fit two categories equally well) and broad enough to cover most personal/world events without constantly needing new ones.

## 🌍 GLOBAL Categories (Macro-History & World Events)

These categories focus on shifts in geopolitics, human knowledge, and global systems.

- **conflict** — armed conflicts, civil wars, and violent uprisings.
  *Examples: World War II, the Syrian Civil War, the Russia-Ukraine War, the French Revolution.*
- **politics** — elections, treaties, geopolitical shifts, and policy changes that don't involve immediate violence.
  *Examples: the Brexit vote, the fall of the Berlin Wall, the creation of the UN, the abolition of Apartheid.*
- **crisis** — economic collapses, natural disasters, and pandemics.
  *Examples: the 2008 financial crisis, the COVID-19 pandemic, the Chernobyl disaster, the Great Depression.*
- **science** — medical breakthroughs, physics, space exploration, and tech inventions.
  *Examples: the Moon landing (1969), the discovery of the DNA structure, the launch of ChatGPT, the first COVID vaccine.*
- **culture** — arts, entertainment, sports milestones, and social movements that shape global consciousness.
  *Examples: the release of the first iPhone, the Beatles on Ed Sullivan, the FIFA World Cup, the #MeToo movement.*

## 🧬 LOCAL Categories (Micro-History & Personal Milestones)

These categories map the standard stages and events of a human lifecycle, focusing on family, career, mobility, and social connections.

- **milestone** — foundational biological and familial events.
  *Examples: birth, a sibling's birth, getting a pet, a major surgery/recovery.*
- **education** — the entire learning journey.
  *Examples: started high school, college graduation, failed an exam, completed a PhD, got a driver's license.*
- **career** — professional life and financial milestones.
  *Examples: first part-time job, laid off, promoted to manager, started a business, quit a toxic job.*
- **relationship** — romantic and close familial shifts.
  *Examples: first kiss, moved in together, wedding day, divorce finalized, breakup.*
- **residence** — separate from travel, since moving is a major life event distinct from visiting.
  *Examples: moved out of parents' house, rented first apartment, bought a house, evicted, moved to a new city/country.*
- **travel** — trips, vacations, and temporary visits.
  *Examples: backpacking through Asia, summer vacation in Greece, first time on an airplane, a road trip with friends.*
- **social** — shifts in your immediate social circle.
  *Examples: met best friend, fell out with a friend, joined a book club, attended a major reunion.*

Each category maps to a single, lowercase CSS class (`.cat-conflict`, `.cat-science`, `.cat-career`, etc.) for consistent, predictable styling. Any category name outside this list still renders correctly under a neutral `other` style instead of breaking.

# 🔒 Security

This app is fully client-side and offline-first — nothing you type or paste is ever sent anywhere.

- **No `innerHTML` with user data.** The renderer builds every timeline card and legend entry using `document.createElement` / `textContent`, never by concatenating user-supplied text into an HTML string. This means pasting malicious input (e.g. text containing `<script>` tags) into the textarea cannot execute as code.
- **Category names are validated** against a strict whitelist before being used as a CSS class, preventing attribute/markup injection through a crafted `CAT:` value.
- **A strict Content-Security-Policy** (`default-src 'self'`) is set via a `<meta>` tag, blocking any external script, style, or resource from loading, as defense-in-depth.

# 🛠️ Roadmap & Upcoming Features (Updated 2026)

This project is evolving to use the latest web standards and provide a more automated experience.

- **Automated Global History Scraping** — A dedicated ingestion pipeline (`scripts/fetch_global_events.py`) to maintain an up-to-date repository of major world events from 1800 to the present.
- **Wikipedia API Integration (Auto-Fill)** — a feature where the user inputs only local events, and the app automatically queries the Wikipedia API for the corresponding years to populate the global events section.
- **CSV/Google Sheets Import** — ability to upload a `.csv` file or connect a Google Sheet to automatically populate the timeline, removing the need for manual text parsing.

- **Advanced Global Filtering** — Real-time filtering by date range and category (Politics, Science, Crisis, etc.) using high-performance DOM reconciliation.
- **Modern Web API Integration**:
    - **Popover API**: For non-modal, lightweight event detail overlays.
    - **CSS Nesting & Container Queries**: Ensuring optimal readability and performance across all devices.
    - **View Transitions API**: Smooth visual state changes when filtering.
- **Data Persistence** — Full implementation of local browser storage for personal timelines.
- **Causal Linking (Impact Arrows)** — visual dotted lines connecting a global event (e.g. "2020 COVID-19") to a local event (e.g. "2020 Lost job"), explicitly showing cause-and-effect relationships.
- **Emoji Icon Parser** — auto-detection of emojis in the `TEXT` field to replace standard text category badges with intuitive icons (✈️ for travel, 🎓 for school, etc.).
- **Dynamic Year Scaling** — adjusting the horizontal width of year-blocks based on event density (a year with 5 events takes up more visual space than a year with 1 event).

- **Light/dark mode in CSS and UI
- **JS file splitting in more files to separate the parsing and other procedures, with versioning. Optimization: Refactored JS to a class-based structure, used DocumentFragment for efficient rendering, and decoupled global events from local input.
- **Security: Performed vulnerability analysis. Confirmed XSS protection. Added data persistence via localStorage
- **Features: Add global event filtering (year range and categories), Popover API for event details, and modern CSS (Nesting, Variables).
