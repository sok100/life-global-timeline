# Life vs. Global Timeline Visualizer

A highly customizable, dual-axis web application that maps personal life events against global macro-historical events. It provides a unique visual perspective on how world history intersects with individual milestones.
The application uses a purely client-side architecture (HTML/CSS/JS) meaning no data is sent to external servers. Events are parsed from a structured text format and rendered dynamically into a responsive, color-coded timeline.

# 🚀 Quick Start
No build tools (Node, npm, Webpack) are required.

Open the index.html file directly in any modern web browser (Chrome, Firefox, Safari, Edge).

# 📝 How to Format Data
The timeline is generated by parsing a specific text structure inside the textarea. Use the following format for each event:

YEAR: [Year] |DATE: [DD.MM] | TYPE: [LOCAL/GLOBAL] | CAT: [Category] | TEXT: [Event Description]
Supported Categories:

As an expert in data structuring and ontology design, I would refine and expand the categories using a standardized nomenclature (lowercase, snake_case or single words work best for code parsing).

When mapping a human life against global history, you need categories that are mutually exclusive (an event shouldn't fit perfectly into two categories) and comprehensive (they should cover 99% of use cases).

Here is the optimal taxonomy I would implement for this project:

# 🌍 GLOBAL Categories (Macro-History & World Events)
These categories focus on shifts in geopolitics, human knowledge, and global systems.

conflict (armed conflicts, civil wars, and violent uprisings).
Examples: World War II, The Syrian Civil War, Russia-Ukraine War, The French Revolution.
politics ( covers elections, treaties, geopolitical shifts, and policy changes that don't involve immediate violence).
Examples: Brexit vote, Fall of the Berlin Wall, Creation of the UN, Abolition of Apartheid.
crisis (Covers economic collapses, natural disasters, and pandemics).
Examples: 2008 Financial Crisis, COVID-19 Pandemic, Chernobyl Disaster, The Great Depression.
science (covers medical breakthroughs, physics, space exploration, and tech inventions).
Examples: Moon Landing (1969), Discovery of DNA Structure, Launch of ChatGPT, First COVID Vaccine.
culture (covers arts, entertainment, sports milestones, and social movements that shape global consciousness).
Examples: Release of the first iPhone, The Beatles on Ed Sullivan, FIFA World Cup, #MeToo Movement.

# 🧬 LOCAL Categories (Micro-History & Personal Milestones)
These categories map the standard stages and events of a human lifecycle, focusing on family, career, mobility, and social connections.

milestone (the foundational biological and familial events).
Examples: Birth, Sibling's birth, Getting a pet, Major surgery/recovery.
education (covers the entire learning journey).
Examples: Started high school, College graduation, Failed an exam, Completed a PhD, Obtained a driver's license.
career (covers professional life and financial milestones).
Examples: First part-time job, Laid off, Promoted to Manager, Started a business, Quit a toxic job.
relationship (covers romantic and close familial shifts).
Examples: First kiss, Moved in together, Wedding day, Divorce finalized, Breakup.
residence (Separated from travel - moving is a major life event, different from visiting).
Examples: Moved out of parents' house, Rented first apartment, Bought a house, Evicted, Moved to a new city/country.
travel (Covers trips, vacations, and temporary visits).
Examples: Backpacking through Asia, Summer vacation in Greece, First time on an airplane, Road trip with friends.
social (covers shifts in your immediate social circle).
Examples: Met best friend, Fell out with a friend, Joined a book club, Attended a major reunion.

# 💡 Why this structure is better for your code:
Uniformity: Every category is a single, lowercase word. This makes the CSS class naming (.cat-conflict, .cat-science, .cat-career) extremely clean.
No Overlap: By merging "Wars" and "Revolts" into conflict, and "Independence" into politics, you remove the user's guesswork. (Is the American Revolution a revolt or independenta? Now it's clearly politics or conflict depending on the year).
Scalability: If a user wants to add a highly specific personal event (e.g., "I won a local marathon"), social or milestone fits perfectly without needing to create a new category just for that.
If you update your index.html file with these exact English keywords in the CSS variables and the parser logic, your timeline will become much more robust and ready for other people to use on GitHub!

# 🛠️ Roadmap & Upcoming Features
This is the MVP (Minimum Viable Product). The following features are planned for future releases:

 CSV/Google Sheets Import: Ability to upload a .csv file or connect a Google Sheet to automatically populate the timeline, removing the need for manual text parsing.
 Wikipedia API Integration (Auto-Fill): A feature where the user inputs only local events. The app will automatically query the Wikipedia API for the corresponding years and populate the Global events section.
 Causal Linking (Impact Arrows): Visual dotted lines connecting a Global event (e.g., "2020 COVID-19") to a Local event (e.g., "2020 Lost job"), explicitly showing cause-and-effect relationships.
 Dynamic Year Scaling: Adjusting the horizontal width of year-blocks based on event density (e.g., a year with 5 events takes up more visual space than a year with 1 event).
 Emoji Icon Parser: Auto-detection of emojis in the TEXT field to replace standard text category badges with intuitive icons (✈️ for travel, 🎓 for school, etc.).
